
# AI-Generated Text Detection Project

This project aims to detect whether a given text is generated by AI or written by a human. It uses a Ko-ELECTRA model for classification.

- **`data/`**: Contains the raw data files (`train.csv`, `test.csv`, `sample_submission.csv`).
- **`models/`**: Stores local model checkpoints (if trained). Pre-trained models are hosted on Hugging Face: `218jw/best_Ada_model`, `218jw/best_extra_model`, `218jw/koelectra_42_4000_model`, and `218jw/koelectra_9000_model`.
- **`notebooks/`**: Contains Jupyter notebooks used for experimentation.
- **`processed_data/`**: Stores the processed and tokenized datasets.
- **`src/`**: Contains all the source code.
  - `data_preparation.py`: A script to preprocess and tokenize the raw data.
  - `electra.py`: A specific training script for the Ko-ELECTRA model with token length filtering.
  - `ensemble.py`: A script to ensemble multiple prediction CSVs using soft voting.
  - `inference.py`: A script to run inference on the test data using a trained model.
  - `modeling.py`: Contains custom model definitions, such as `SuperLossElectraForSequenceClassification`.
  - `superloss.py`: Implementation of the SuperLoss function.
  - `train.py`: A generic script for training the model with various configurations.
  - `train_9000.py`: A training script variant for the full 9000 dataset without fixing the random seed.
  - `train_super_ada.py`: A specific training script converted from the `3rd super_Ada.ipynb` notebook.
- **`ko_README.md`**: The Korean version of this documentation.
- **`README.md`**: This file.
- **`requirements.txt`**: Contains python dependencies.

## Setup

1.  **Install dependencies:**

    ```bash
    pip install -r requirements.txt
    ```

    *(Note: A `requirements.txt` file should be created based on the project's dependencies.)*

2.  **Place the training data:**

    Ensure that `train.csv` is located in the `data/` directory.

## How to Run

### 1. Data Preparation

First, you need to preprocess the raw data. This script will read `data/train.csv`, split it into training and evaluation sets, tokenize the texts, and save the result in the `processed_data/` directory.

```bash
python src/data_preparation.py \
    --input_csv data/train.csv \
    --output_dir processed_data \
    --dataset_name tokenized_ds
```

### 2. Training

Once the data is prepared, you can train the model. The `train.py` script can be used for general-purpose training. You can enable `SuperLoss` by adding the `--use_superloss` flag.

**Standard Training:**

```bash
python src/train.py \
    --output_dir models/my_electra_model \
    --processed_data_path processed_data/tokenized_ds
```

**Training with SuperLoss:**

```bash
python src/train.py \
    --output_dir models/my_superloss_model \
    --processed_data_path processed_data/tokenized_ds \
    --use_superloss
```

Alternatively, you can run the specific training configuration from the original notebook:

```bash
python src/train_super_ada.py
```

### 3. Inference

After training, you can use the `inference.py` script to generate predictions on the test set. You can provide a local checkpoint path or a Hugging Face model hub path (e.g., `218jw/koelectra_9000_model`).

```bash
python src/inference.py \
    --checkpoint 218jw/koelectra_9000_model \
    --test_csv data/test.csv \
    --output submission
```

This will create a `submission.csv` file in the root directory.

### 4. Ensemble

If you have inference results (CSVs) from multiple models (e.g., Ada, Extra, etc.), you can ensemble them by averaging their prediction probabilities (Soft Voting) using the `ensemble.py` script.

```bash
python src/ensemble.py \
    --csvs sub_ada.csv sub_extra.csv sub_42_4000.csv sub_9000.csv \
    --output final_ensemble_submission.csv
```

Once completed, a `final_ensemble_submission.csv` file will be created as the final ensemble result.
